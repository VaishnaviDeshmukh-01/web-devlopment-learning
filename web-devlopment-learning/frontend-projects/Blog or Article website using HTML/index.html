Introduction

HTML (HyperText Markup Language) is the standard markup language used to create and structure content on the web. It is the foundation of every webpage, using a system of tags and elements to tell a web browser how to display text, images, links, and other media. W3Schools W3Schools +3 HTML is a markup language, not a programming language, meaning it uses annotations (tags) to define the structure and layout of content, not logic or functionality. When a browser receives an HTML document, it reads the markup and renders the content visually for the user. GeeksforGeeks GeeksforGeeks +3

Key components of HTML include:

  1. Tags: Keywords enclosed in angle brackets (e.g., <p>), which typically come in pairs (opening and closing tags) to delineate elements.

  2. Elements: Consist of the start tag, content, and end tag (e.g., <p>This is a paragraph.</p>).

  3. Attributes: Provide additional information about an element, specified within the start tag as name="value" pairs.
Together with CSS (Cascading Style Sheets) for styling and JavaScript for interactivity, HTML forms the core of modern web development.

History of HTML

HTML (HyperText Markup Language) was invented by physicist Tim Berners-Lee in 1991 at CERN to allow researchers to share documents. Originally designed with 18 basic tags, it has evolved from a simple text-formatting tool (HTML 1.0, 1993) to the sophisticated, multimedia-capable standard of HTML5 (2014) that powers the modern web.

HTML Attributes

HTML attributes are special words used within the opening tag of an element to provide additional information or properties. They typically come in name="value" pairs and are used to define the element's behavior, appearance, or functionality.


  • Placement: Attributes are always included within the start tag of an HTML element.

  • Syntax: They are written as name-value pairs, for example, <a href="https://example.com">.

  • Quoting Values: The W3C recommends always using quotes for attribute values to avoid errors, especially if the value contains spaces or special characters.

  • Case Sensitivity: While HTML itself is largely case-insensitive, the W3C recommends using lowercase for attributes and their values for consistency.

  • Types: Attributes can be general (global, applicable to most elements) or element-specific (only relevant to certain elements).

Common Element-Specific Attributes

Common HTML element-specific attributes include src and alt for media (images/videos), href for links, and type, value, min/max for form inputs. Other key attributes are style for styling, class and id for identification, and title for tooltips.
